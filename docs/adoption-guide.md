# Adoption Guide

## 導入レベル
- Level 1: GitHub中心（個人向け）
- Level 2: Linear + Notion併用（チーム向け）
- Level 3: MCP運用・監査証跡・品質ゲート必須（商用/監査向け）

## 導入手順
1. `README.md` と `docs/core/development-protocol.md` を確認
2. 対象レベルを選択
3. Linearプロジェクト/Issue運用を設定
4. Notionの作業ログDBを準備
5. PRテンプレート、CODEOWNERS、workflowを有効化
6. 最初のIssueで試験運用

## 最小運用
- Level 1はLinear/Notionを推奨（任意）
- Level 2以上はLinear/Notion連携を標準運用
- Level 3はMCP利用ログと品質ゲート完了を必須化
