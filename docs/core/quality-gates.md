# Quality Gates

各ゲートは 目的 / 確認項目 / 必須エビデンス / 完了条件 / NG例 を持つ。

## 1. 要件明確化ゲート
- 目的: 手戻り防止。
- 確認項目: 目的、受け入れ条件、非対象。
- 必須エビデンス: Linear Issue更新履歴。
- 完了条件: 人間レビュー済み。
- NG例: 受け入れ条件なし着手。

## 2. 設計レビューゲート
- 目的: 設計妥当性確認。
- 確認項目: 代替案、影響範囲。
- 必須エビデンス: 設計メモ/ADR。
- 完了条件: 合意済み。
- NG例: 主要判断の記録なし。
