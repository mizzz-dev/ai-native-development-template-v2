# Security Policy

## Vulnerability reporting
脆弱性は**公開Issueに投稿しないでください**。

- GitHub Security Advisories (Private vulnerability reporting) の利用を推奨
- secret / token / credential / private URL を貼らない

## Supported versions
- main branch の最新
- 最新の安定リリース系列

## Expected response
- 受付確認: 3営業日以内を目標
- 初期トリアージ: 7営業日以内を目標
- 修正方針: 影響範囲評価後に案内

## Safe disclosure guideline
- PoC共有は最小限
- 再現手順は悪用困難な形で記述
- 公開は修正提供後に調整
